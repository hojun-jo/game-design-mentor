# Game Design Mentor

초보 인디 개발자를 위한 웹 기반 게임 기획 리뷰 MVP다.

## 프로젝트 소개

Game Design Mentor는 자유형 게임 기획 초안을 받아 초보 인디 개발자가 **무엇을 먼저 결정하고 검증해야 하는지** 찾도록 돕는 AI 멘토링 도구다. 아이디어의 좋고 나쁨을 단정하거나 완성된 기획을 대신 작성하지 않는다. 대신 기획 의도, 대상 플레이어, 핵심 반복 구조, MVP 범위를 함께 점검하고, 사용자가 다음 기획에서도 활용할 수 있는 판단 기준을 제공한다.

### 해결하려는 문제

첫 게임을 기획할 때는 아이디어보다 우선순위를 정하는 일이 어렵다. 어떤 정보가 빠졌는지, 핵심 루프가 실제로 반복할 만한지, 기능 중 무엇을 먼저 만들어야 하는지 판단하기 쉽지 않다. 감상 위주의 피드백은 이러한 결정을 도와주지 못하는 경우가 많다.

이 프로젝트는 다음 질문에 답할 수 있도록 설계됐다.

- 이 게임은 **누구에게 어떤 감정**을 주려는가?
- 플레이어는 무엇을 반복하고, 어떤 피드백과 보상 때문에 다시 행동하는가?
- 차별점은 문구가 아니라 실제 플레이 경험으로 이어지는가?
- 첫 프로토타입에서 반드시 검증할 가설과 과감히 제외할 기능은 무엇인가?
- 플레이테스트에서 관찰하거나 물어봐야 할 것은 무엇인가?

### 대상 사용자와 사용 방식

주요 대상은 게임 개발과 기획이 아직 익숙하지 않은 초보 인디 개발자다. 사용자는 채팅에 게임 콘셉트, 목표 플레이어, 전달하고 싶은 감정, 반복 플레이 구조, 주요 기능, 참고 게임 등을 자유롭게 적거나 Markdown 기획 파일을 업로드한다. 정해진 양식에 맞춰 처음부터 작성할 필요는 없다.

앱은 자유형 입력을 구조화한 뒤, 리뷰에 꼭 필요한 `의도한 감정`과 `코어 루프`가 있는지 확인한다. 정보가 부족하면 곧바로 평가를 시작하지 않고 보완 질문으로 전환한다. 답변이 누적되면 다시 기획을 읽어 충분한 시점에 리뷰를 진행한다.

### 리뷰 내용

리뷰는 다음 세 가지 축을 중심으로 진행된다.

- **의도 정렬**: 목표 플레이어와 원하는 경험이 게임 콘셉트 및 참고 게임과 일관되는지 확인한다.
- **코어 루프와 차별점**: 플레이어의 반복 행동, 피드백, 보상, 다음 선택이 명확한지 살피고 차별점이 실제 경험으로 이어지는지 검토한다.
- **MVP 범위와 플레이테스트**: 가장 먼저 검증할 가설을 중심으로 구현 범위를 줄이고, 관찰 가능한 플레이어 행동과 질문으로 테스트 계획을 만든다.

각 축의 결과에는 짧은 진단뿐 아니라 판단에 사용한 근거와 설계 원칙을 함께 담는다. 또한 상반된 **두 가지 해석 방향**을 근거와 트레이드오프와 함께 비교해, 사용자가 하나의 정답 대신 자신의 목표에 맞는 선택을 할 수 있게 한다.

### 주요 기능

- **자유형 초안 구조화**: LLM이 비정형 기획 텍스트를 콘셉트, 타깃, 감정 목표, 코어 루프, 기능, 보상 구조, MVP 목표 등의 공통 형식으로 정리한다.
- **단계적 보완 대화**: 하드 필수값이 없거나 권장 정보가 부족하면, 누락 항목만 묻는 대신 판단에 도움이 되는 질문을 제시한다.
- **레퍼런스 조사**: 입력된 참고 게임과 자동으로 찾은 유사 게임의 공개 정보를 확인해 리뷰 근거로 활용하고, 출처를 함께 제공한다.
- **MVP와 테스트 계획**: 구현 범위를 줄일 수 있는 제안, 가장 먼저 검증할 가설, 테스트 대상과 플레이테스트 질문을 정리한다.
- **조건부 엔진 추천**: 플랫폼, 시각 표현, 네트워크 요구, 팀 경험 등의 정보가 충분할 때만 엔진 후보를 비교하고, 정보가 부족하면 필요한 기술 조건을 먼저 묻는다.
- **리뷰 후속 대화**: 리뷰의 해석 오류를 정정하거나 판단 근거를 질문할 수 있다. 정정 내용은 새 리뷰 생성에 반영된다.

### 리뷰 결과물

한 번의 리뷰가 끝나면 사용자는 다음 결과를 확인할 수 있다.

- 의도 정렬, 코어 루프, 범위/플레이테스트에 대한 진단과 근거
- 각 리뷰에 적용한 게임 디자인 원칙과 보완 질문
- 서로 다른 두 방향의 선택지, 장점, 비용
- 우선 제외하거나 축소할 기능과 첫 플레이테스트 가설
- 다음 기획에서도 재사용할 수 있는 자기 점검 질문
- 지금 가장 먼저 결정할 사항을 요약한 한 줄 결론

### 동작 흐름

`기획 초안 입력 → 구조화 추출 → 게임 기획 관련성 확인 → 필수/권장 정보 검증 → 보완 대화 또는 레퍼런스 조사 → 3개 리뷰 축 분석 → 두 방향 비교 → 학습 요약 및 후속 대화`

현재 구현 범위는 `LangGraph 기반 자유형 초안 입력 -> LLM 구조화 추출 -> 필수/권장 입력 검증 -> OpenAI web search 기반 유사 레퍼런스 게임 자동 발굴 및 공개 정보 조회 -> 보완 채팅 -> 3개 리뷰 축 -> 2개 방향 비교 -> 최종 요약 -> 리뷰 후속 대화`이다.

### 기술 구성

- **UI**: Streamlit
- **워크플로 오케스트레이션**: LangGraph
- **LLM 연동 및 레퍼런스 검색**: OpenAI API와 web search
- **언어 및 패키지 관리**: Python 3.13+, uv

### MVP 범위와 제한 사항

현재 버전은 빠른 1회성 교육형 리뷰를 검증하기 위한 MVP다. 기획 저장소, 버전 이력, 실시간 공동 편집, 플레이 영상 자동 분석, 플레이테스트 회고의 장기 누적은 포함하지 않는다. 레퍼런스 정보와 엔진 관련 선택은 의사결정을 돕는 출발점이며, 출시 정책·라이선스·가격·지원 플랫폼 같은 최신 사실은 최종 결정 전에 공식 문서로 확인해야 한다.

## 현재 사용자 흐름

1. 본문 채팅에 기획 초안을 붙여넣거나 사이드바에서 `.md` 파일을 업로드한다.
2. LLM이 기획 초안을 구조화한다.
3. 앱 코드가 `emotion_goal`, `core_loop` 하드 필수값과 권장 입력을 검증한다.
4. 리뷰 가능 상태가 되면 OpenAI web search로 유사 레퍼런스 게임을 최대 3개 자동 발굴하고, 사용자가 입력한 레퍼런스와 함께 공개 정보 및 citation을 재검증한다.
5. 하드 필수값 또는 권장 입력이 비면 보완 채팅으로 전환하고, 답변을 반영해 다시 구조화한다.
6. 리뷰 가능 상태가 되면 의도 정렬, 코어 루프, MVP 범위/플레이테스트 계획을 짧게 진단한다.
7. 정확히 2개의 해석 방향과 각각의 근거, 트레이드오프를 보여준다.
8. 지금 먼저 정해야 할 것 한 줄로 마무리한다.
9. 리뷰 완료 후 본문은 후속 대화 채팅으로 쓰고, 리뷰 리포트와 요약은 접이식 영역과 사이드바에서 확인한다.
10. 사용자는 후속 대화에서 해석 오류를 정정하거나 리뷰 근거를 질문할 수 있다.
11. 정정 메시지는 원문에 누적되어 리뷰를 다시 생성하고, 단순 질문은 현재 리뷰 컨텍스트로 답변한다.
